# 仿生记忆评测框架(BMB)v2 设计文档

> 一句话:测的不是「信息恢复多少」,而是**一个记忆系统的外部行为有多接近健康成人记忆的实证分布**。

---

## 0. 为什么要重做(v2 与 v1 的根本不同)

v1 的评测目标是**信息压缩效率**——一个探针,地板(空载荷)到天花板(完整真相态),架构的载荷能把分数推多高就值多少,沿预算画「保真–预算前沿」。它名副其实地只测了「压缩」这一面。

但项目叫 **Biomimetic**(仿生)。仿生记忆远不止压缩:情绪会加权、信念会被后到信息修正、回忆时会合理重构、信息会随时间遗忘、与自我相关的会被偏置保留、而且回忆本身会重写记忆(再巩固)。v1 把这些统统排除(理由是「测不到 / 不可复现」),于是名实不符。

v2 的纠偏是根本性的:**评测 ≠ 实现。** 框架不实现任何仿生记忆系统,只定义「什么样的记忆系统算仿生」(行为规约),再测量任意系统对这个规约的对齐度。于是所有「测不到所以砍掉」的逻辑垮掉——测不到的是系统内部机制,但生物记忆的**外部行为**有认知科学实证数据,系统对不对齐这些数据,可以量。

v2 不是 v1 的升级,是换地基:从「测信息效率」换成「测仿生行为对齐度」。v1 只有几项**工程模式**(确定性生成、自检硬门、黑盒接口)作为真遗产保留,评分范式和维度集合从头来——v1 的 `ProblemFamily`、`recovery_ratio`、budget 评分轴、scoring 函数**全部 replaced,不降级、不沿用**。

---

## 1. 第一性原理:评测目标

### 1.1 目标定义

**评测目标 = 系统行为对齐「健康成人记忆」的实证分布。**

「健康成人记忆」是关键限定——把病理(阿尔茨海默晚期的大规模虚构)排除在外。系统的外部行为分布(保留率随时间怎么变、细节漂移怎么分布、情绪信息是否被优先保留、召回是否改写后续记忆……)越接近健康成人记忆的实证分布,对齐度越高。

### 1.2 评分是钟形,不是单调

由此推出评分形状:**钟形(有最优区间),不是单调递增。**

- 完全不遗忘的冷库 → 不对齐生物 → 低分。
- 永不漂移的逐字存储 → 不对齐 → 低分。
- 但漂移成纯幻觉、遗忘到丢核心事实 → 同样不对齐**健康**分布 → 也低分。

「过度」和「不足」都扣分,只有落在健康区间才高分。这天然区分「健康生物特征」与「病理」,也是 v2 优于「最大化复刻人脑」方案的根本原因——后者无法区分合理重构与虚构幻觉,会让纯幻觉系统拿高分,逻辑上站不住。

### 1.3 为何排除另外两个方向

- **「只借鉴有用机制」**(把遗忘 / 漂移当缺陷不测):标尺退化回工程性能,跟 v1 精神没区别,且名实不符。
- **「最大化复刻人脑」**(缺陷越多越像):无法区分健康再构与病理虚构,一旦引入「健康 / 病理」边界就退化成本框架的钟形对齐。逻辑上无法独立成立。

### 1.4 标尺的客观性

标尺来自认知科学实证(艾宾浩斯遗忘曲线、情绪增强记忆效应量、再构记忆漂移分布、自我参照效应等),不是某个系统的输出。跨系统、有学术背书,是框架公信力与「适合当行业标准」的来源。

## 2. 维度全集(身份边界)

### 2.1 筛选原则

维度 = **健康成人记忆必然有、而冷库(数据库 / 向量库,最不仿生的参照系)必然没有**的外部行为特征。每个维度都直接拉开「仿生 vs 冷库」,不为可测性妥协。这是「一步到位」的落点:框架定义一次性完整覆盖仿生记忆。

### 2.2 六个维度

| 维度 | 一句话定义 | 认知科学数据硬度 |
|---|---|---|
| **遗忘曲线** | 保留率随时间衰减,形态对齐艾宾浩斯 | 最硬 |
| **情绪增强** | 带情绪色彩的信息系统性优先保留 | 硬 |
| **再构 / 再巩固** | 多次回忆同一事件细节合理重组(再构);且召回动作本身会改写记忆、影响后续召回(再巩固) | 硬 / 中 |
| **压缩 / 图式** | 记要点、丢细节、用结构存 | 中 |
| **信念更新** | 后到的矛盾信息修正旧信念,旧态不顽固残留 | 中偏脏 |
| **自我关联** | 与主体相关的信息优先保留 | 软 |

六个维度全要,构成框架完整定义。「再构 / 再巩固」是一个维度测两层(见 §4),因为它俩共享「多次召回」的探测协议,只是测量信号不同。

## 3. 核心抽象:可控实验环境

### 3.1 导演 vs 被试

框架把评测建模成一场**可控实验**:
- **框架 = 导演**:知道全部真相,持有每个事实的生物权重,持有生物基准分布。
- **adapter(被测系统)= 被试**:只看到 ingest 进来的事件流,对真相结构和基准保密。

### 3.2 adapter 契约(从第一性原理重设,非 v1 扩展)

```python
class BMBAdapter(Protocol):
    def ingest(self, user_id: str, ts: float, event: StructuredEvent) -> None: ...
    def recall(self, user_id: str, cue: str, current_ts: float,
               session_id: str | None = None) -> str: ...
```

- `ingest(user_id, ts, event)` —— **在线、按逻辑时间顺序调用**:导演按 `ts` 升序逐条(或分批)喂入事件,模拟记忆的实时形成。`ts` 是逻辑时间戳(导演注入,非墙钟)。`event` 结构化,携带:事实 ID、情绪强度(valence × arousal)、自我关联度、以及「被哪条后续事件修正」的指针。
- `recall(user_id, cue, current_ts, session_id) -> str` —— 在逻辑时间 `current_ts`、线索 `cue` 下召回;`session_id` 标记对同一目标的反复召回。**recall 是有副作用的操作**:它模拟「回忆」这个动作,允许(甚至期望,对再巩固维度)修改 adapter 的内部状态,从而影响同一 `session_id` 的后续 recall。不是无状态读取。

**没有 budget 参数。** adapter 返回多少记忆表征,由它自己的容量管理决定——这本身就是它的记忆能力(压缩能力正体现在此)。框架层面有一个可选的全局输出上限(`max_recall_tokens`,默认关闭)做工程保护,但它是配置项,不是 adapter 契约的一部分,更不是评分轴。

### 3.3 逻辑时间是硬前提

逻辑时间由导演注入,不是墙钟。不可绕过:要测遗忘曲线就必须能控制时间,评测不可能等几个月真实流逝。所有被测系统都得接受「可注入时间」的接口——但见 §8,这是评测激励,不是接入障碍。

### 3.4 评测单元(显式定义)

一个维度的完整评测 = 一次**受控实验**,由四部分组成:

1. **合成世界**:导演生成的一张事实图,每个事实带已知生物权重(§7)。
2. **探测协议**:该维度规定的探测方式——在哪些 `current_ts`、用哪些 `cue`、召回几次(§4)。
3. **行为样本**:adapter 在探测协议下产生的召回输出序列。
4. **基准比对**:从行为样本里提取该维度的行为信号(保留率 / 漂移量 / 偏置……),与健康成人记忆的基准分布算对齐度(§5)。

这套取代了 v1 的「单探针 → 单次 assemble → 单点恢复率」。一个维度是一条**行为分布**,不是一个点。

## 4. 六个测量透镜

每个维度共用 §3 的 ingest/recall,差异在探测协议和提取的信号。

| 维度 | 探测协议 | 提取的信号 | 生物基准 |
|---|---|---|---|
| 遗忘曲线 | 同一事实,在多个 `current_ts` recall | 保留率(记不记得:yes/no)vs 逻辑时间 | 艾宾浩斯曲线 |
| 情绪增强 | 对称事实对(情绪 / 中性),同时间 recall | 保留偏置(情绪保留率 − 中性保留率) | 情绪增强效应量 |
| 再构 / 再巩固 | 同一事实,同 `session_id` 多次 recall | (a) 输出漂移分布;(b) 第 k 次 recall 是否被第 k−1 次改写 | (a) 健康再构漂移分布;(b) 再巩固效应量 |
| 压缩 / 图式 | 长流水账 recall 要义 | 要义事实保留 + 表面细节丢弃比 | 图式记忆衰减比 |
| 信念更新 | 按时序 ingest 旧 → 矛盾新 → recall | 旧态被修正的程度(旧事实是否还残留) | 信念修正更新度 |
| 自我关联 | 对称事实对(相关 / 无关),recall | 保留偏置 | 自我参照效应量 |

**关键性质:`current_ts` 和 `session_id` 是 recall 的参数,不是接入门槛。** 系统用它们做衰减 / 重构,就测出对应行为;忽略它们(纯语义 RAG、无状态召回),就在该维度表现为冷库 → 低对齐度。这是正确的评测结果,不是接入失败。

**信号提取的确定性原则(见 §5.2):** 上表里的信号——「记不记得某事实」「旧事实是否残留」「漂移量」——尽量由**确定性事实探针**直接产出(yes/no 或数值),而不是靠 LLM judge 从自由文本里猜。

## 5. 评分:对齐度

### 5.1 单维度对齐度

每个维度,把 adapter 的行为信号(一条曲线,如「保留率 vs 时间」;或一个标量,如「情绪 / 中性保留差」;或一个分布,如「漂移量分布」)与该维度的生物基准算距离 → 归一化到对齐度 ∈ [0, 1]。钟形:落在健康区间高分,两端(冷库 / 病理)低分。

### 5.2 信号提取:确定性优先(评分器作为一等公民)

v2 测的是精细行为分布,评分噪声会直接污染「可复现的行业标准」。所以信号提取遵循:

- **确定性事实探针优先**:探针设计成能让信号直接以 yes/no 或数值产出。例如遗忘曲线——探针就是「系统在 t 时刻还记不记得事实 F?」,由确定性匹配判定(事实 F 的关键标识是否出现在召回里),不请 LLM 当裁判。
- **LLM judge 最小化**:只在无法避免时(如再构漂移的「合理程度」判断)用,且用多次采样投票降低方差,并记录方差供使用者审视。
- **评分器独立设计**:不沿用 v1 的 `scoring.py`(那是为信息恢复率写的);v2 的评分器按「行为信号提取」重新设计,每个透镜一个。

### 5.3 分项为主,可选总分

- **分项(核心输出)**:6 个维度各给对齐度 ∈ [0, 1]。透明、可解释,维度异质(曲线 vs 偏置 vs 分布),强制合成会丢信息。
- **可选总分(便利)**:等权聚合,供排名 / 传播;使用者可自定义权重重新聚合。

默认输出 = 6 个分项 + 一个等权总分。

## 6. 自检电路(继承 v1 精神,重定义两端)

每次出分前跑硬门,两端不正常则结果作废。

- **地板 = 冷库 adapter**(全保真、不衰减、不漂移、不修正)→ 应在六维都拿低分(不对齐生物)。冷库同时是「最不仿生参照系」。
- **天花板 = bio-faithful adapter**(按生物基准分布生成行为的模拟器)→ 应拿高分。
- **硬门**:天花板总对齐度必须显著高于地板,否则测量不可信,结果作废(`SelfCheckError`)。

自检验证的是「评分器能否区分最不像生物 vs 最像生物」——这是通用好实践(v1 的地板/天花板精神),但两端的具体定义按 v2 的对齐度重设。

---

## 7. 数据生成(state-first,在线时序)

继承 v1 的状态优先生成范式,**确定性、可复现**,但强化两点:结构化生物权重 + 在线时序喂入。

1. 导演先生成**结构化真相**:一张事实图,每个事实带生物权重(情绪强度、自我关联度、时间戳、被哪条后续事实修正)。
2. 按逻辑时间顺序**渲染成事件流**,**在线**喂给 adapter(`ingest` 按 `ts` 升序调用),模拟记忆实时形成。
3. ground truth 持有完整真相;adapter 对真相结构保密。
4. 探针由结构化真相**确定性派生**(遗忘曲线探针 = 选定事实 + 一组 `current_ts` 采样点;情绪增强探针 = 选定一对情绪 / 中性事实 + 同时间 recall;信念更新探针 = 选定一对矛盾事实 + 按时序 ingest)。

事件流可按 adapter 的能力声明(§8.2),选择把标注作为结构化字段传入,或渲染进自然语言文本——两套测的是不同能力,都 valid。

## 8. adapter 接入(onboarding)

框架能否成为行业标准的关键。核心:**接入门槛最低 + 包容异构 + 可比 + 引导进化。**

### 8.1 最小契约

adapter 只实现 `ingest` + `recall` 两个方法(§3.2)。接入的全部就这些。其余(探针生成、生物基准、对齐度计算、信号提取)都是框架的事。

### 8.2 能力声明

adapter 声明它支持什么,框架据此标注每个维度是「真测了」还是「因不支持而冷库化」:

- 接受**结构化 event** 还是只吃**纯文本**(后者:标注渲染进自然语言,测系统能否从文本隐式识别情绪 / 自我关联并加权——本身是 valid 能力);
- 用不用 `current_ts`(时间衰减);
- 用不用 `session_id` 且 recall 是否有副作用(可重复召回 / 再巩固)。

透明标注。使用者在榜单上能看到「这个系统在遗忘维度是真测了,还是因不支持时间衰减而冷库化」。

### 8.3 分级榜单

- **完整仿生榜**:声明全能力的系统,6 维全测,进总榜(含总分)。
- **基础榜**:只支持基础能力(无时间 / 无副作用召回)的系统,只在它能真测的维度(压缩、情绪、自我关联)上比。

异构系统都有位置,同级别内可比。这同时是引导力:想在完整榜拿名次,就得支持时间衰减和有副作用的召回——框架用参数定义了「完整仿生记忆需要什么能力」。

### 8.4 多绑定(语言 / 隔离)

底层一个 **JSON 标准协议**(ingest / recall 请求-响应 schema),上面提供多种绑定:

- **Python SDK**(主绑定);
- **CLI / 子进程绑定**(非 Python 或想进程隔离);
- **HTTP 绑定**(远程 / 服务化系统)。

### 8.5 参考实现

框架自带样板 adapter:冷库(地板)、真相(oracle,参考)、bio-faithful(天花板),外加 1–2 个真实系统样板(纯向量 RAG、LLM 摘要记忆),带能力声明示范。

## 9. 与 v1 的关系(replaced,不是降级)

| v1 资产 | v2 处置 |
|---|---|
| 信息恢复率 `recovery_ratio` 评分范式 | **replaced** → 对齐度(§5) |
| `ProblemFamily`(A/C/B 三难题) | **replaced,彻底丢弃** → 六维透镜(§2、§4)。压缩能力不继承 A/C/B 的测量,压缩 / 图式维度从「图式化对齐生物」重新定义 |
| budget 评分轴(保真–预算前沿、AUF) | **replaced,彻底丢弃** → adapter 自管容量,budget 不再存在(§3.2),仅留可选全局输出上限做工程保护 |
| `scoring.py`(关键词 + LLM judge) | **replaced** → 确定性事实探针优先的信号提取器(§5.2) |
| adapter 黑盒契约 | **真遗产,保留** → ingest/recall(§3.2),但签名重设(在线时序、有副作用 recall、无 budget) |
| 自检电路(地板 / 天花板硬门) | **真遗产,保留精神** → 冷库地板 / bio-faithful 天花板(§6),按对齐度重定义两端 |
| state-first 确定性数据生成 | **真遗产,保留** → 强化结构化生物权重 + 在线时序喂入(§7) |

v1 不白做:它验证过的工程模式(黑盒契约、自检硬门、确定性生成)是 v2 地基;但评分范式、维度集合、budget 轴、scoring 全部 replaced,不沿用一行旧逻辑。

## 10. 已知风险与待解

诚实标出未完全敲定的点,不在 spec 里藏成「TBD」:

1. **生物基准的获取与干净度** —— 认知科学曲线多是特定实验、特定材料下测的,抽象成普适标尺有解释空间。哪些够硬(艾宾浩斯、情绪增强)、哪些偏软(自我关联、再巩固效应量)需一轮考据。每个维度的基准须附文献来源与适用边界。
2. **信号提取器的设计** —— 确定性事实探针的「关键标识匹配」要做到既不漏判也不误判;再构漂移的「合理程度」无法完全避开 LLM judge,须用多次采样 + 方差披露控制噪声(§5.2)。这是各透镜的独立工程,也是框架最吃创造力的部分。
3. **再巩固的可测性** —— 「第 k 次 recall 是否被第 k−1 次改写」需要一个对照实验(有 / 无前置 recall 的后续 recall 差异),协议设计比其他维度复杂,基准数据也偏软。
4. **六维全实现的工作量** —— 定义一步到位(本文档),实现按维度推进(实现规划阶段排期),数据硬的维度(遗忘、情绪、再构)先行,软的(自我关联、再巩固)接口先留。
5. **结构化 event 渲染策略对测量的影响** —— 同一标注,作为结构化字段 vs 渲染进文本,测的是不同能力(§8.2)。两套都 valid,但须在榜单上区分,避免跨能力对比失真。
