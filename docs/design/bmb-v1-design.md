## 0. 摘要（TL;DR）

BMB 是一个评测**记忆架构**（memory architecture）的基准。它不测"答案准不准"，而测：

> **一个记忆架构，在固定回答模型 + 硬预算 B 下，能用多小的"记忆载荷"恢复出这段过去的多少决策相关信息。**

分数是 `[0,1]` 区间的「**信息恢复率**」，沿预算 B 画成「**保真–预算前沿**」。

一句话核心：**BMB 测的是各记忆架构逼近"最小充分统计量"的程度。**

---

## 1. 背景与动机

工业界主流记忆测试（AMB、MemGPT-bench 等）停留在"机械式查表"：喂事实 → 提问 → 算召回率。它有三个根本缺陷：

1. 只考物理召回，不考"聪明地遗忘 / 凝练 / 改信念"；
2. **"傻存储 + 聪明回答 LLM" 就能拿高分**——测的是 LLM 推理，不是记忆；
3. 缺可复现性（模型漂移、数据不冻结）。

BMB 的目标是建立**记忆架构的能力标准**：可复现、不可作弊、能区分"是记忆模块在干活还是 LLM 在硬推"。

若维护者同时是参赛方，这种利益相关靠**全程开源 + 冻结数据集 + 评分代码公开**对冲——所有指标都基于公开、可复现、不可作弊的测量，不依赖任何主观裁定。

**v1 目标**：一条可信、可复现、可对外发表的"记忆架构"赛道。对话级 Agent（ChatGPT/Gemini 记忆等）赛道留作后续，v1 不做。

## 2. 第一性原理推导

### 2.1 记忆模块的四个原语
一个记忆模块本质上只做四件事：**选存（DECIDE）→ 凝练（CONSOLIDATE）→ 修正（REVISE）→ 召回（RETRIEVE）**。

### 2.2 维度"够格"的四条判据
一个被测维度必须同时满足：

1. **落在记忆层**——不能溜到编排层 / LLM 推理 / Agent 主动性；
2. **正交**——捕获别的维度盖不住的失效模式，不重复计分；
3. **API 级可测**——能通过 `ingest`/`assemble` 探到，打分站得住；
4. **不可替代**——拿掉它，"记忆好坏"出现明显盲区。

### 2.3 用判据审旧"四维"

| 旧维度 | ①记忆层 | ②正交 | ③可测 | 判决 |
|---|---|---|---|---|
| A 衰减/显著性 | ✅ | ✅ | ✅ 确定性 | 留 |
| B 认知抽象 | ✅ | ✅ | ⚠️ LLM-judge | 留 |
| C 矛盾自愈/时序 | ✅ | ✅ | ✅ 确定性 | 留 |
| D 情绪/联想唤醒 | ⚠️ 混入 Agent 主动性 | ⚠️ 与召回通道重叠 | ❌ | **砍** |

**D 的三宗罪**：① "下雨→主动问情绪"测的是 Agent 主动性，而 API 级 `query` 是被动的，测不到；② 召回不是与选存/凝练/修正并列的第四种能力，是**观测另外三个的唯一通道**，单列必重复计分；③ 几乎没有真实模块给记忆打"情绪 tag"，硬测只能靠 LLM 猜，违背可复现。

### 2.4 那把刀：把"记忆能力"和"LLM 聪明"分开

> **一个"傻存储 + 聪明回答 LLM"就能通过的测试，测的是 LLM，不是记忆。**

只看"最终答案对不对"，傻库 + 聪明 LLM 能蒙混 A/B/C 全部：矛盾靠 LLM 看时间线解、抽象靠 LLM 推、噪音靠 LLM 读全量。**所以"答案准确率"根本不是记忆指标。**

### 2.5 真正的核心：最小充分统计量 × 预算

隔离"架构"与"模型"的唯一办法：**固定回答模型 + 硬预算 B**。架构交出 ≤B 的载荷，固定模型凭它作答。

- 准 + 载荷**小** → 记忆架构在干活（筛选 / 修正 / 凝练）；
- 准 + 载荷**巨大** → 是回答 LLM 在海量 dump 里硬推，**判负**。

载荷的"理想形态"在信息论里有精确名字：**关于这段历史、对回答该问题充分的最小统计量（minimal sufficient statistic）**。BMB 测的就是各架构逼近它的程度。

### 2.6 范式跃迁

- **旧范式**："喂经历 → 提问 → 给答案打分（裸准确率）"——继承自 AMB，未经审视。
- **新范式**：测载荷的「**信息恢复率**」，沿预算画前沿，以真相状态为天花板。提问降级为"**探针**"（测量仪器），不再是被测对象。

理论原理（艾宾浩斯 / 图式 / 信念修正 / 赫布）与测量维度**不必一一对应**：原理是"看世界的眼镜"，本设计落地为「**1 个核心指标 × 3 个压缩难题 × 预算前沿**」。

---

## 3. 核心定义

**范式一句话**：架构的 ≤B 载荷，保住了这段过去的多少决策相关信息——用恢复率 ∈ `[0,1]` 量，沿预算 B 画前沿。

**三个压缩难题**（从"为什么会逼近不了充分统计量"推出，各自独立、各自是一种失效模式）：

- **A 噪音→显著性**：必须剪掉 99% 废话、留住那 1% 救命信息（如青霉素过敏）；
- **C 历史→当前态**：必须让新事实**覆盖**旧事实，不能把整条时间线都背着；
- **B 流水账→隐结构**：必须从散落事件**抽象**出潜在画像（如数码党）。

（砍掉的 D 不在此列。）

## 4. 测量与打分

### 4.1 单探针 × 单预算

对探针 `q`、预算 `B`、架构 `A`：

1. `载荷 = A.assemble(user, q, B)`（≤B，自包含）
2. `架构分 s_arch = score(fixedLLM(q, 载荷), 真相答案) ∈ [0,1]`
3. `地板 s_floor = score(fixedLLM(q, ""), 真相答案)`（空载荷："啥都不给能蒙对多少"）
4. `恢复率 R = clamp((s_arch − s_floor) / (1 − s_floor), 0, 1)`

> **分母为什么是 `(1 − s_floor)`**：天花板是 1（完美答案）；恢复率 = "从'啥都不知道'到'完美答对'，架构走完了多少"。
> **零信号过滤（探针质量门）**：另算 `s_ceiling = score(fixedLLM(q, 真相状态), 真相答案)`。若 `s_ceiling − s_floor < ε`（天花板没比地板高 = 无区分度——可能是题太简单 `s_floor≈1`、或题坏/模型太弱 `s_ceiling≈s_floor`），**剔除这道探针**。存活探针满足 `s_ceiling≈1` 且 `s_floor<1`，故 `(1−s_floor)` 是良定义分母、恢复率 ∈[0,1]。`s_ceiling` 只用于过滤，不进恢复率公式。

> 例子：地板=0.8（题简单）、架构分=0.9 → 恢复率=0.5。简单题里那 10% 的真实提升被正确识别为"只恢复了一半信息"，不被高分掩盖。难度混淆变量就此消掉。

### 4.2 聚合

- **（架构 × 难题 × 预算）**：该难题下所有探针的恢复率取均值 → 前沿上一个点；
- **前沿**：每架构 × 每难题，画"恢复率–预算"曲线，预算档 `{8k, 32k, 64k, 128k}`；
- **榜单标量**：**前沿下面积 AUF**（area-under-frontier）；曲线本身做诊断 / 雷达。

### 4.3 打分函数 `score`

- **A/C**（事实型 / 时序型）→ **确定性**（真相是已知标签，匹配或语义判定）；
- **B**（开放式推荐）→ 固定模型当 judge + **多次采样投票**，报 CI。

承袭"确定性优先"共识：能确定性就确定性，只有天然开放式的 B 才用 LLM-judge。

### 4.4 反作弊闭环

固定模型人人相同 + 载荷硬卡 ≤B + 真相状态对架构不可见 + 零信号探针被滤——架构唯一能拉开差距的就是**同等预算下载荷的信息量**。没缝可钻。

### 4.5 固定模型（测量仪器）

全文的"固定模型"指**同一个 pinned 开源模型**（锁 snapshot + seed + temp=0），同时承担"作答"（4.1）和"B 难题判分"（4.3）。它是**测量仪器**，不是参赛方，对所有架构同一把尺。

- 载荷 token 用**该模型的 tokenizer** 计；
- 该模型上下文窗口须 ≥ 最大预算档（128k），才能吃下最大载荷；
- 数据**渲染**（5.3）用的模型是另一回事，记在数据集 manifest 里（可与测量模型不同，渲染模型不参与测量）。

## 5. 数据生成（state-first）

### 5.1 原理：状态先行，渲染在后

1. **作者相（结构化真相）**：seeded RNG 直接编出"真相状态"——哪些救命事实、状态变量怎么翻转、潜在画像是什么、配多少噪音。**真相是结构化、精确、机读的。**
2. **渲染表相（经历流）**：结构化真相 + 噪音 → 带时间戳的自然对话。这步才用 LLM，但它是**纯渲染**——真相已定死，LLM 再发挥也改不了 ground truth；且逐事件渲染、上下文小、不会崩、可并发。
3. **派生探针**：探针与标准答案**从结构化真相确定性产出**，不经 LLM 标注，杜绝标注错误。

> **真相是规格，流是渲染产物，探针是规格的投影。** 三者天然自洽。

### 5.2 三难题构造

| 难题 | 结构化真相 | 渲染成的流 | 探针 / 标准答案 |
|---|---|---|---|
| **A 噪音→显著性** | 一组"救命事实"+ 注入时刻（day312：青霉素过敏） | 99% 程序化废话 + 1% 事实自然嵌入 | "这用户有什么用药禁忌？" → 那条事实（确定性判分） |
| **C 历史→当前态** | 状态变量时间线（diet：vegan@0→破戒@12→复素@30） | 每次状态变化的自然对话 | "用户现在饮食？" → 时间线**最后一次**（确定性判分） |
| **B 流水账→隐结构** | 潜在画像 + 支持事件（键盘/GPU/Github ⇒ 数码党），画像从不直说 | 散落事件自然化 | "送什么生日礼物？" → 画像 + 可接受答案集（LLM-judge） |

三者的"真相状态"都很小（就是充分统计量本身）——正好给 4.1 当天花板验证用。

### 5.3 生成器：确定性管线，不是 agent 操作

v1 数据集由一条**确定性 Python 管线**产出：

```
python generate.py --seed 42 --config v1.yaml
  → 结构化真相 + 渲染流 + 探针/答案
```

- 结构层 = seeded RNG（完全确定）；
- 渲染层 = 固定模型 + temp=0 + 固定 seed（可复现）；
- **冻结数据集本身由管线产出**，不经 agent 手（agent-in-the-loop 不可复现）。

### 5.4 冻结与分发（可复现契约）

- 生成**一次** → 把（结构化真相 + 渲染流 + 探针/答案）整体打快照、加版本号 `v1`、不可变，随仓库分发；
- 榜单分数**绑定数据集版本**报（`@v1`）；
- 生成器一并开源：别人能复跑验证或扩新版本，但 v1 这个文件就是铁打的标尺（MMLU 套路：考卷冻结、方法公开）。

### 5.5 v1 规模（克制，可调）

- ~30 个虚拟用户，每人 ~5万–15万 token 的流（够让"暴力 dump"超过最小预算 8k）；
- 噪音比 ≈ 99%（A）、状态翻转 2–4 次（C）、支持事件 8–15 个（B）；
- 全量冻结数据集几百万 token 级，生成与评测都扛得住。
- **不碰 README 里"10M/十年"的远期目标——v1 要的是可信，不是噱头规模。**

## 6. 适配器契约

### 6.1 接口

```python
class BMBMemoryArchitecture(Protocol):
    def __init__(self, config: dict): ...

    def ingest(self, user_id: str, ts: str, text: str) -> None:
        """按时间序流入一条经历。同一 user_id 的多次 ingest = 该用户完整记忆。
        user_id 之间相互隔离。内部若有随机性，必须用固定 seed。"""

    def assemble(self, user_id: str, query: str, budget_tokens: int) -> str:
        """交出一份 ≤budget_tokens、自包含的记忆载荷（纯文本）。
        载荷格式自由（散文/列表/JSON 都行）——怎么排版让固定模型最好用，是架构的"空间规划"能力。
        返回后架构不再被调用；固定模型只凭这份载荷作答。"""
```

关键设计：`assemble() -> 载荷` 只交载荷、**答案永远来自固定模型**（避免架构与回答模型混淆，从而把"记忆能力"和"LLM 聪明"分开）；预算作为入参，同题跑 4 档自然生成曲线。

### 6.2 硬规（harness 强制，不信适配器自觉）

- **预算强制**：harness 数 token；返回 >B 的，**硬截断到 B 并记录**（v1 默认策略；让"把最重要的放前面"本身成为能力，超出部分丢弃）。
- **作答时零检索**：`assemble` 返回后，架构不得再被调用、不得回调任何外部检索——载荷即全部。
- **确定性**：同 ingest 序列 + 同 config → 同输出（内置 seed）。
- **用户隔离**：`user_id` 是独立记忆空间，不得串。

### 6.3 两个内置参考适配器（不需要任何真实 API，让框架自洽）

1. **`NaiveDumpAdapter`（地板标尺）**：append 全部；`assemble` 时返回最近 / 关键词命中的原始切片凑满预算。零记忆智能。作用：跑通管线（冒烟）、经验地板（被反超 = 架构不如不归纳）、验证探针够狠（它若小预算还高分 = 题太简单，警报）。
2. **`OracleAdapter`（天花板标尺，合法作弊）**：绕过 ingest，直接拿**真相状态**当载荷返回。作用：验证测量正确（必须拿≈天花板分；拿不到 = 固定模型或探针坏了），把恢复率 `[0,1]` 两端用真跑的数钉死。

> 这俩是框架的**自检电路**。换新数据集 / 新固定模型，先跑 naive + oracle，两端正常再跑被测架构。

### 6.4 真实系统以后插

Mem0 / Letta / Zep / 任意自研记忆系统，都只是再写一个实现上面 Protocol 的类 + 一份 config，配一份 `docs/adapter_authoring.md`。

**v1 只交付接口 + 两个参考适配器，不接任何真实 API。** 接口在，谁来都能插。

> 可选 hook（v1 不实现）：`assemble` 可附带返回"本次内部消耗的 token / 算力"，给将来的"架构开销"旁支轴用。接口先开好，不填。

## 7. 可复现性与仓库结构

### 7.1 复现契约（"同一个数"）

三个输入全部钉死 → 结果（几乎）逐位相同：

- **数据集版本**：v1 冻结、不可变、随仓库分发；分数永远带 `@v1`。
- **固定模型**：锁到具体 snapshot + seed + temp=0（开源、可自托管）。
- **依赖**：pinned（`requirements.txt` / lockfile）。

唯一剩的方差源 = **B 难题的开放式判分** → k 次采样投票压到紧置信区间，结果里**报 CI**。其余全部确定性。

### 7.2 一条命令

```bash
python run_benchmark.py --dataset v1 --adapter naive_dump
```
产出 `results/<adapter>@v1-<model>-<seed>.json`：每个架构 × 每压缩难题 × 每档预算的恢复率 + 前沿曲线 + AUF + 完整配置快照。

### 7.3 自检硬门（写进流程，不是可选）

每次出分前，harness **先跑 naive + oracle**：naive 必须在低端、oracle 必须在天花板。**不通过 = 这次结果作废、不报。** 把第 6.3 的自检从"建议"变成流水线里的硬门。

## 8. v1 范围

**做（v1 交付）**：适配器接口契约；两个内置参考适配器（naive / oracle）；确定性生成器 + 冻结 v1 数据集；测量管线（恢复率 / 前沿 / AUF / 天花板验证）；自检硬门；一条命令复现；本设计文档。

**不做（明确延后）**：对话级 Agent 赛道；D 情绪联想；真实系统适配器；架构开销旁支轴；CI 冒烟；"10M/十年"规模。

## 9. 术语对照

| 通用 / 旧称 | 本范式 |
|---|---|
| 召回率 / recall | **信息恢复率** |
| 判卷 / Judge | **测量**（固定模型当 judge 是其中一环） |
| 维度（A/B/C/D 四维） | **压缩难题**（A/C/B 三个） |
| 突击测验 / 考题 | **探针** |
| ground truth 考卷 | **真相状态 + 探针答案** |
| token 惩罚 | **预算 B**（考规，不是事后惩罚） |

## 10. 已知限制与后续（接真实模型前）

v1 随 `FakeModel` 发货、smoke 数据集自证可跑。以下在**接入真实开源模型 / 生成 `datasets/v1` 前**补齐（终审记录，非阻塞）：

- **自检绝对阈值**：`self_check` 当前只验"oracle AUF > naive AUF"（相对）；补"oracle ≈ 天花板、naive ≈ 地板"的可配绝对阈值（按族），防止"测量仪器本身已坏却仍通过相对门"。
- **B 族报 CI**：B 难题是唯一方差源，`score_llm_judge` 需返回均值 + 样本/CI 并透出到结果 JSON（§4.3/§7.1）。
- **结果/manifest 记模型 snapshot**：把固定模型的 `id / snapshot / seed / temperature` 写进 manifest 与结果 JSON，完整可复现契约（§7.1）。
- **适配器构造契约**：`BMBMemoryArchitecture` 显式声明 `__init__(config: dict)` + `@runtime_checkable`；CLI 加 `--adapter-config`，为接入真实系统铺路。
- **真 tokenizer**：`count_tokens` 换成固定模型的真 tokenizer（现为 `len//4` 占位，全链路一致故自洽）。
- **预算截断记录 + 二分**：截断时记日志（现静默），并改二分（现逐字符 O(n²)）。
- **C 族多关键词/语义兜底**：C 难题 accept 当前单关键词子串匹配，真实模型换说法易漏判，补多关键词或语义兜底。
- **smoke 仅锻炼 A 族**：`FakeModel` 简单响应器总先匹到 A 族词，C/B 在 smoke 下无信号（结果里以 `no_signal_families` 如实标出）；真实模型下三族皆有信号。
